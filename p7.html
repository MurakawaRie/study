연습문제 part 7

1. 다음은 HTTP 상태코드에 대한 내용입니다
    1-1. 200
  • 서버가 요청을 제대로 처리했음
    1-2. 404
  • Not Found, 서버가 요청한 페이지를 찾을 수 없음
    1-3. 405
  • 허용되지 않은 메소드에 접근하였음
    1-4. 500
  • 내부 서버 오류, 서버오류로 요청을 수행할 수 없는 상태
2. 제네릭(Generic)
    2-1. 제네릭이란 무엇입니까?
  • 데이터의 타입을 일반화(Generalize) 시켜주는 것
    2-2. 제네릭을 사용할시에 장점은?
  • 클래스나 메소드 내부의 객체 타입 안정성을 높일 수 있다
    반환값에 대한 타입변환이나 타입검사에 들이는 노력을 줄일 수 있다
3. 컬렉션 프레임워크
    3-1. 자바에 있어서 '컬렉션 프레임워크'는 무엇입니까?
  • 다수의 데이터를 쉽고 효율적으로 처리할 수 있게 해주는 클래스의 집합이다
    (인터페이스를 사용해서 구현한다)
    3-2. 자바에 왜 '컬렉션 프레임워크'가 생겼을까요
  • 코드를 직접 작성할 필요 없이 이미 구현된 컬렉션 클래스를 목적에 맞게 사용할 수 있게 해줌
    3-3. 빈칸에 알맞은 내용을 넣으세요
  • 자바 컬렉션은 (객체)를 수집해서 저장하는 역할을 한다
    프레임워크란 (데이터를 처리하는 알고리즘)을 미리 정해놓은 (클래스)이다
    컬렉션의 주요 (인터페이스)로는 (List),(Set),(Map)가 있다
    3-4. 빈칸에 알맞은 내용을 넣으세요
  • (ArrayList),(Vector),(LinkedList)는 List인터페이스를 구현한 클래스로
    List인터페이스로 사용 가능한 컬렉션이다
    3-5. 빈칸에 알맞은 내용을 넣으세요
  • (HashMap),(TreeMap),(Hashtable)는 Map인터페이스를 구현한 클래스로
    Map인터페이스로 사용 가능한 컬렉션이다
    또한 Map인터페이스는 제네릭 타입이다
    3-6. List 컬렉션은 어떤 방식으로 객체를 담고 사용합니까?
  • 배열을 이용하여 객체를 담고 같은 요소의 중복 저장도 허용함
    3-7. 빈번한 객체 삭제와 삽입이 일어나는 곳에서는 이것을 사용하는것이 좋다
  • LinkedList
    3-8. List 인터페이스는 무슨 타입입니까?
  • 제네릭
    3-9. ArrayList가 3-7과 비교해 장점을 서술하세요
  • 데이터를 읽어오거나 저장할때 효율적이다
    데이터의 저장순서를 유지하고 Object 배열을 이용해 순차적으로 저장하기 때문
    3-10. 빈칸에 알맞은 내용을 넣으세요
  • Map 컬렉션은 ( 키(key) )와 ( 값(value) )으로 구성된 Entry 객체를 저장하는 구조를 가지고 있다
    여기서 키와 값은 모두 ( 객체 ) 이다
    또한 Map 컬렉션은 키와 값의 중복저장이 ( 불가능 )하다
4. HashMap
    4-1. 빈칸에 알맞은 내용을 넣으세요
  • HashMap은 Map인터페이스를 구현한 대표적인 Map 컬렉션이다
    주로 키 타입은 ( String )을 사용한다
    4-2. 빈칸에 알맞은 내용을 넣으세요
  • 키와 값의 타입은 ( 배열 )을 사용할 수 없고
    클래스와 인터페이스만 사용이 가능하다
5. JSP
    5-1. DTO를 왜 만들어야 합니까?
  • 비즈니스 로직의 캡슐화로 보안을 강화할 수 있다
    5-2. getter , setter는 왜 만드는 것입니까?
  • 직접적인 접근을 막고 외부에서 메소드를 통해 접근하도록 유도하기 위해서
    5-3. default 생성자를 만들지 않을 경우 무슨 일이 발생합니까?
  • 컴파일러가 자동으로 default생성자를 추가해준다
    5-4. DAO는 왜 만들어야 합니까?
  • 효율적인 커넥션 관리와 보안, DB접근을 위한 로직과 비즈니스로직을 분리하려고
