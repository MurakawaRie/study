8. Java의 일부분입니다. (이게 참 중요함)
String a = "12345" 가 있고 String b = new String("12345") 가 있습니다

a.equals(b) = ? 답이 뭘까요?
equals (메소드)의 경우 : true -> 내용 자체를 비교함

a == b 로 했을 시 답이뭘까요?
==(비교연산자)의 경우 : false -> 대상의 주소값(메모리 참조값)을 비교함

대상의 주소값이 달라지는 이유는 String b는 새로운 객체가 생성되어 또다른 String으로 저장이 된다
이를 알기 위해서
System.out.println(System.identityHashCode(a));
System.out.println(System.identityHashCode(b));
를 사용하면 HashCode 값이 달라짐을 확인 할 수 있다

- StringBuffer : String과 달리 가변성을 가짐(문자열 연산이 많고 단일스레드이거나 동기화를 고려하지 않을때)

10. java의 접근 제어자에는 public, protected, default , private 가 있다. 각각의 종류와 특징을 서술해 보시오
- public : 프로젝트 안이라면 어디에서나 직접 접근 가능
- private : 선언한 클래스를 제외하고 외부에서 직접 접근 불가능(getter setter를 이용)
- protected : 같은 패키지에 포함되어있거나, 직접 상속받은 외부 패키지에서만 접근 가능
- default : 별도로 지정되지 않을 경우 자동적으로 가지게 되는 접근 제어자(같은 패키지에서만 접근가능)
