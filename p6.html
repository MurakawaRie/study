연습문제 part 6

1. 다음은 배열(Array)에 대한 질문입니다
    1-1. 배열에 대해 정의하세요
  • 배열은 동일한 자료형의 데이터를 연속된 공간에 저장하기 위한 자료구조다
    1-2. 배열의 index에 대해 정의하세요
  • 배열의 요소마다 붙여진 일련번호로서 각 요소를 구별하는데 사용한다
    1-3. 다음의 빈칸에 알맞은 말을 넣으세요
  • 배열에 요소를 넣지 않을 경우 숫자 자료형은 (0)
    문자 자료형은 (null) 로 초기화 된다
2. 다음은 ArrayList 에 대한 질문입니다
    1-1. ArrayList에 대해 정의하세요
  • List인터페이스의 제너릭 클래스를 배열 형태로 관리할 수 있게 해주는 클래스
    순차적으로 자료를 정리하여 관리할 수 있다
    1-2. ArrayList와 배열(Array)의 차이는?
  • 길이를 조절할 수 있는가에 대한 여부
    Array(고정) vs ArrayList(변경 가능)

    primitive type(기본 자료형)의 사용가능 여부
    Array(가능) vs ArrayList(Object만 가능)
3. 다음은 클래스(Class)에 대한 질문입니다
    3-1. 자바에서 클래스(Class)를 사용하는 이유에 대해 서술하세요
  • 객체(Object)를 만들기 위해서
    3-2. 메소드(Method)란 무엇입니까?
  • 어떠한 특정 작업을 수행하기 위한 명령문의 집합
4. 접근 제어자
    4-1. 접근제어자는 무엇입니까?
  • 클래스와 클래스멤버의 선언 시 부가적인 의미를 부여하는 키워드
    4-2. 접근 제어자는 4종류가 있는데 각각의 명칭과 설명을 하세요
  • - private : 외부에서 접근 불가능한 제어자
    - public : 외부에서 접근 가능한 제어자, 해당 객체를 사용하는 프로그램 어디서나 접근가능
    - default : 기본적으로 설정되는 제어자, 같은 패키지에 속하는 멤버나 클래스만 접근가능
    - protected : 부모클래스는 접근이 가능하나, 외부에선 접근이 불가능한 제어자
    4-3. 캡슐화에 대해 설명하세요
  • 변수로의 직접 접근을 막고 꼭 필요한 정보와 기능만 외부에 오픈하는것
    4-4. 캡슐화에 대한 예로 우리가 사용하는 방식은 어떤것이 대표적입니까?
  • DTO(Data Transfer Object) 클래스
5. 생성자(Constructor)
    5-1. 생성자란 무엇입니까?
  • 객체의 생성과 동시에 인스턴스 변수를 원하는 값으로 초기화 할 수 있는 메소드
    5-2. 모든 클래스는 기본 생성자를 가지고 있습니까?
  • 가지고 있다
    단 사용자가 별도로 사용자 정의 생성자를 만들면 사라진다
6. 상속(Inheritance)
    6-1. 상속의 장점은 무엇이 있습니까?
  • 중복된 코드를 줄이고 유지보수가 편리하고 통일성이 있음
    6-2. 상속을 여러개 하는것이 가능합니까?
  • 가능하다. 부모가 여러 자식 클래스로 상속할 수 있다
    6-3. super()란 무엇입니까?
  • 자식 클래스가 부모 클래스로부터 상속 받은 멤버를 참조할때 사용하는 참조변수
    6-4. 상속 받지 않은 독자적 클래스가 자동으로 상속받는 이 클래스는?
  • Object
7. 인터페이스(Interface)
    7-1. 인터페이스란 무엇입니까?
  • 클래스를 동일한 목적하에 동일한 기능을 수행하도록 하는것
    7-2. 인터페이스의 장점은 무엇입니까?
  • 개발코드 수정을 줄이고 프로그램 유지 보수성을 높일 수 있다
    7-3. 인터페이스와 클래스의 차이점은 무엇입니까?
  • 인터페이스 : 상수와 추상메서드만 가질 수 있음
    클래스 : 일반 변수나 생성자,추상메서드 모두 가질 수 있음
    7-4. 인터페이스를 다른 클래스나 인터페이스에 사용할때 무엇이라고 얘기하는가
  • 구현
    인터페이스에 정의된 멤버들을 클래스에서 대신 '구현' 하는것
    인터페이스는 구현을 상속과 같이 할 수 있다.
8. 오버라이딩(Override)
    8-1. 오버라이딩은 무엇입니까?
  • 부모 클래스가 가진 메서드를 하위 클래스에서도 상속되어 사용할 수 있게 하는것
    8-2. 이것이 붙은 내용은 오버라이딩이 불가능하다 이 2가지는?
  • static : 정적메소드이기 때문
    final : 더 이상 상속이 불가능한 클래스가 됨
    8-3. 오버로딩(Overloading)과 오버라이딩(Overriding)의 차이점은?
  • -오버로딩 : 파라미터 개수가 다르거나 자료형이 달라야 사용가능
    -오버라이딩 : 파라미터 개수와 자료형이 같아야 사용가능
    8-4. 오버라이딩의 장점은?
  • 하나의 코드로 다양한 객체를 처리할 수 있다
    8-5. 다형성에 대해 서술하세요
  • 하나의 객체나 메소드가 여러가지 다른 형태를 가질 수 있는것
    (오버로딩, 오버라이딩, 상속받은 개체의 참조변수 형변환 등)
